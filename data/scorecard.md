# AI Engineering Discipline Scorecard

Use this scorecard weekly during the pilot. Score each item from 0 to 2.

- `0`: absent or unreliable
- `1`: partially adopted
- `2`: consistently adopted

## Spec

| Item | Score |
|---|---|
| Requirements have acceptance criteria |  |
| Non-goals are explicit |  |
| Cross-module changes have design docs or ADRs |  |
| Requirements map to tests |  |

## Verify

| Item | Score |
|---|---|
| CI blocks failing tests |  |
| PRs include verification evidence |  |
| AI usage is disclosed in PRs |  |
| Review checks edge cases and rollback |  |

## Memory

| Item | Score |
|---|---|
| Project rules are documented |  |
| Module boundaries are documented |  |
| Repeated review issues become memory entries |  |
| Incidents update pitfalls or ADRs |  |

## Interpretation

- `0-8`: AI coding is mostly unmanaged.
- `9-16`: partial discipline exists, but quality depends on individuals.
- `17-24`: team has a repeatable AI-assisted development system.
